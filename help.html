产品介绍

NetTermPro 是一款功能强大的多协议终端工具，支持多种网络协议和本地终端访问。无论是远程服务器管理还是本地命令行操作，NetTermPro 都能为您提供稳定、高效的终端体验。

SSH 连接

支持 SSH2 协议，支持密钥认证和密码认证，安全可靠。

Telnet 连接

标准 Telnet 协议支持，适合网络设备管理。

串口通信

全参数支持，包括波特率、数据位、校验位、停止位、流控。

FTP 客户端

支持文件上传下载，目录管理。

本地终端

支持 CMD 和 PowerShell，使用 ConPTY 技术。

快速开始

第一步：创建连接

  1. 点击菜单栏 文件 > 新建连接
  2. 在弹出的对话框中选择协议类型
  3. 填写连接参数（主机地址、端口、用户名等）
  4. 点击 连接 按钮

第二步：使用终端

  • 连接成功后，终端会显示远程主机的欢迎信息
  • 直接输入命令并按回车执行
  • 使用鼠标滚轮滚动查看历史输出

第三步：保存连接

点击 文件 > 保存连接，下次可以从 最近连接 菜单快速访问。

提示： 连接参数会自动保存到程序目录下的 connections.json 文件中。

协议使用指南

1. SSH 连接

SSH（Secure Shell）是加密的远程登录协议，是管理Linux服务器的标准方式。

  • 主机地址： 服务器IP或域名
  • 端口： 默认 22
  • 用户名： 登录用户名
  • 认证方式： 密码或密钥文件
安全提示： 建议使用密钥认证方式，将公钥放在服务器的 ~/.ssh/authorized_keys 文件中。

2. Telnet 连接

Telnet 是传统的远程终端协议，常用于网络设备（如路由器、交换机）的管理。

  • 主机地址： 设备IP地址
  • 端口： 默认 23
注意： Telnet 协议不加密数据传输，不建议在不安全的网络中使用。

3. 串口连接

串口连接用于连接串口设备，如路由器控制台、嵌入式设备等。

  • 端口名称： 如 COM1, COM2, /dev/ttyUSB0
  • 波特率： 常用值 9600, 19200, 38400, 115200
  • 数据位： 通常为 8
  • 校验位： None, Even, Odd
  • 停止位： 1, 1.5, 2
  • 流控： None, RTS/CTS, XON/XOFF

4. FTP 连接

FTP 用于文件传输，支持上传和下载文件。

  • 主机地址： FTP服务器地址
  • 端口： 默认 21
  • 用户名： FTP用户名
  • 密码： FTP密码

5. 本地终端

本地终端提供对系统命令行的访问，支持 CMD 和 PowerShell。

  • 本地CMD： Windows 命令提示符
  • 本地PowerShell： PowerShell 环境
特性： 本地终端使用 ConPTY 技术，完美支持退格、Ctrl+C、top 等命令。

设置说明

点击 工具 > 设置 打开设置对话框，可以配置以下选项：

常规设置

  • 主题： 深色或浅色主题
  • 编码： UTF-8、GBK、GB2312 等
  • 历史大小： 终端历史记录条数

连接设置

  • 连接超时： 连接等待时间（秒）
  • 命令超时： 命令执行超时时间（秒）
  • 自动重连： 连接断开后自动重试

外观设置

  • 字体： 终端显示字体
  • 字体大小： 字体像素大小

行为设置

  • 关闭到托盘： 关闭窗口时最小化到系统托盘
  • 最小化到托盘： 最小化窗口时隐藏到系统托盘

常见问题

为什么连接 SSH 失败？

可能的原因：

  • 服务器地址或端口错误
  • 服务器防火墙阻止了连接
  • 用户名或密码错误
  • SSH 服务未在服务器上运行

请检查以上项目，并确保网络连接正常。

中文显示乱码怎么办？

请尝试以下方法：

  1. 在设置中更改编码格式（UTF-8 或 GBK）
  2. 确保远程服务器的 LANG 环境变量设置正确
  3. 尝试使用 chcp 65001（Windows）或 export LANG=zh_CN.UTF-8（Linux）
串口连接无法建立？

请检查：

  • 串口线是否正确连接
  • 端口名称是否正确（如 COM1, COM2）
  • 波特率是否与设备匹配
  • 是否有其他程序占用了该串口
本地终端无法输入？

本地终端使用 ConPTY 技术，需要 Windows 10 1809 或更高版本。如果遇到问题：

  1. 确保 Windows 版本符合要求
  2. 尝试以管理员身份运行程序
  3. 检查是否有其他终端程序冲突
如何使用密钥认证？

SSH 密钥认证设置步骤：

  1. 生成密钥对：ssh-keygen -t rsa
  2. 将公钥复制到服务器：ssh-copy-id user@host
  3. 在连接对话框中选择密钥认证，并指定私钥文件路径

私钥文件通常位于 ~/.ssh/id_rsa（Linux/macOS）或 C:\Users\用户名\.ssh\id_rsa（Windows）。

连接列表保存在哪里？

连接列表保存在程序目录下的 connections.json 文件中。日志文件保存在 logs/nettermpro.log。

如何导出/导入连接配置？

使用 文件 > 导出连接 和 文件 > 导入连接 菜单可以备份和恢复连接配置。
